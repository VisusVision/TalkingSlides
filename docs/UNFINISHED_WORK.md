# Unfinished Work

This file documents repository features that are incomplete, partially implemented, not wired to UI, configured but inactive, fragile, or still needing verification. Status labels use only the requested labels.

Phase 1 TTS preview is implemented on both the TTS service and Django API. Phase 2 project-level TTS settings persistence is implemented in Django. Phase 3 worker/rerender wiring now passes saved project TTS settings into generation. Phase 4 Studio TTS controls are implemented inside the Studio editor workspace. Phase TTS-D1A deterministic backend resolution is implemented for acronyms, curated English technical fallback, and unknown/ambiguous term metadata; the Django preview fail-open path now runs local D1 normalization instead of returning raw text when the TTS service is unavailable. Phase TTS-D1B Studio preview display is implemented for unknown/ambiguous resolver terms, resolver rules, draft add-override actions, and optional L1B suggestion requests. Phase TTS-L1A backend pronunciation suggestion endpoint, Phase TTS-L1B Studio suggestion UI, and Phase TTS-L1C optional Ollama setup docs are implemented as optional Studio assistance only, disabled by default unless configured, and never render-path intelligence. Phase 5A Studio transcript edit/save/rerender UI is implemented. Phase 5B Studio scene rail, slide summary, timeline selection, and selected-scene transcript focus are implemented. Phase 5C now has backend transcript page actions, Studio controls for split, merge, move up/down reorder, delete, and restore, plus structural rerender sequencing polish that keeps page actions on full same-project rerender. Phase 5 QA hotfixes improved TTS preview language auto-detection, added optional short preview audio, hid technical chunk details by default, enabled draft split-by-blank-lines for the selected page, replaced browser confirmations with in-app confirmation UI, and added Studio editor focus mode with larger transcript editing. Phase 5 hotfix work moved real transcript, slides, notes, and TTS editing into the Studio editor workspace, made lesson detail tabs read-only/summarized, tightened save/refetch behavior, and added regressions for preserving other pages and `original_text`. Worker routing now splits non-avatar render jobs onto a lightweight `render` queue and avatar/MuseTalk/LivePortrait jobs onto an `avatar` queue. Original-language SRT/WebVTT playback is implemented, and subtitle translation now includes the backend data model, provider contract, metadata-only track listing, manual Studio generation, mock-provider SRT/VTT generation, and an automatic API/Ollama/LibreTranslate/Argos/mock provider fallback chain. Phase 6B adds context-aware local LLM cue-batch translation through Ollama while preserving original cue timing. Phase 6D1 adds backend public request guardrails: language allowlist, cache rate limits, per-project/language generation locks, max active generation slots, and production mock fallback safety. The final CC phase adds the Watch user-language request dropdown for missing supported languages and refreshes/selects tokenized tracks after generation; subtitle provider calls now run asynchronously in `worker.tasks.generate_translated_subtitle_track_task` so API requests return 202 instead of waiting for provider I/O. Remaining items focus on curated dictionary expansion, drag-and-drop/richer visual scene editing, and staging validation of production translation providers.

`Needs verification` means repository inspection found some evidence, but not enough to prove the full end-to-end product behavior.
